<2. 캡술화, 상속, 다형성>
* 상속은 어느정도 알고있었음
* 캡슐화, 다형성은 아는데 정확한 설명이 어려웠음

*캡슐화 :
말그대로 캡슐화? 어떤 속성을 캡슐에 넣고...뭘 하는거
=> 데이터를 캡슐로 싸서 외부 접근으로부터 데아터 보호하는 것

*상속 :
부모 클래스의 속성을 자식 클래스에서 사용가능한 것?
=> 자식 클래스의 객체가 생성될때 자식 클래스에 선언된 멤버뿐만 아니라 
부모 클래스에 선언된 멤버도 함께 가질 수 있는 속성

*다형성 :
다양한...속성?
=> 하나의 기능이 상황에 따라 다르게 작동하는 현상




<3. C언어에 추가한 기능>
* 팀원 전부 관련된 내용을 배웠지만 설명을 하기엔 너무 어려움이 있었음
배웠다는 기억만 나는 정도

- 함수 중복 : 매개 변수 개수나 타입이 다른 똑같은 이름의 함수 선언들
- 디폴트 매개 변수 : 매개 변수 값이 전달되지 않는 경우에 디폴트 값이 넘어가도록 함수를 선언하는 거
- 참조와 참조변수 : 변수에 별명을 붙여서 변수공간을 같이 사용할 수 있는 개념
- 참조에 의한 변수 : 함수 호출시 참조를 전달할 수 있게 한다
- new/delete 연산자 : new는 동적메모리 할당 시, delete는 할당된 메모리를 해제할 때 사용하는 것
(*동적 메모리 할당? 프로그래밍 실행 중에 필요한만큼 메모리를 할당하는 기법)
- 연산자 재정의 : 기존 연산자에 새로운 연산을 정의할 수 있게 하는 것
- 제네릭 함수와 클래스 : 함수나 클래스를 데이터 타입에 의존하지 않고 일반화 시킬 수 있게 한 것



<4.> 
* 캡술화 복습
- 클래스는 붕어빵 틀 = 객체 만드는 틀
- 객체는 거기서 만들어진 붕어빵(=실체)
- 클래스 안에 있는 속성들을 캡슐로 싸서 외부로부터 보호하는 특성!

* 상속성 복습
- 상속성은 코드 작성 시 중복된 코드를 작성하지 않고 클래스 사이의 상속관계를 두어서
자식 클래스를 생성할 때 부모 클래스에 선언된 멤버들을 가지고 생성하는 것

*다형성 복습
- 다형성은 하나의 기능이 상황에 따라 다르게 작동하는 현상인데
그 중 함수 재정의(=오버라이딩)는 부모 클래스에 구현된 함수를 동일한 이름으로 자식
클래스에서 다르게 구현하는 것을 말한다.
- 연산자나 함수를 중복해서 사용할 수 있다.
- 하나의 함수를 여러개의 자식 클래스에 상속할 수 있는데(=상속성), 이때 그 함수는 클래스에 따라
각각 다르게 보이거나 작동하는 현상을 다형성이라고 한다.
